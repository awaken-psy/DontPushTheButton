# M2: 最小可玩 Demo（核心循环验证）

**目标**：打通 GDD 核心循环——**配置阶段绑定键位（含魔键）→ 进入关卡 → 用普通/魔键能力过关 → 腐败累积与满格重置**。用 1 个示范房间（对标 GDD「魔键初现」）验证「键位重构」这个机制到底好不好玩。评分系统/多关卡/更多能力一律推到后续。

## 任务清单
- [ ] 2.1 能力系统骨架：定义 `Ability` 基类/接口，把 M1 的移动/跳跃包装成能力，新增「推动(Push)」能力
- [ ] 2.2 按键绑定数据模型：能力↔按键映射、槽位限制（默认 4）、普通键 vs 魔键标记
- [ ] 2.3 关卡配置（ScriptableObject）：每关定义可用按键集合、魔键、可用能力、槽位数
- [ ] 2.4 配置阶段 UI（控制面板）：拖拽能力图标到可用按键、显示槽位占用、魔键高亮、「开始关卡」按钮；未绑定移动则拦截
- [ ] 2.5 运行时动态重绑：进入关卡时按配置把能力挂到对应按键；魔键上的能力触发强化效果（最小版：强化跳跃）
- [ ] 2.6 魔键系统：按下魔键 → 强化能力 + 腐败值增加
- [ ] 2.7 腐败系统最小版：HUD 腐败条、满格(100)→关卡重置回配置阶段、屏幕边缘泛红初版反馈
- [ ] 2.8 示范房间：对标 GDD「魔键初现」——一条宽沟；魔键=空格强化跳跃可过，普通跳跃需找箱子垫脚
- [ ] 2.9 关卡流程串联：配置 → 进入 → 抵达出口（胜利 UI）/ 腐败满格（重置）；GameManager 接管状态机
- [ ] 2.10 核心循环试玩测试 + 数值手感粗调（腐败增量、强化倍率、跳跃距离）

## 备注
- 这是「核心乐趣验证」里程碑：宁可系统简陋，也要让"是否用魔键"的取舍真正可感。
