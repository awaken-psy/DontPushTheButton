# M1: 工程骨架与核心移动

**目标**：Unity 2022.3 LTS + URP 工程初始化、git 协作就绪；第三人称角色能在白盒测试房间里移动和跳跃。产出第一个能跑的 build——这是技术底座验证，不涉及键位重构核心玩法。

## 任务清单
- [ ] 1.1 初始化 Unity 工程（URP 模板）+ git init + Unity 官方 `.gitignore` + 首次提交
- [ ] 1.2 项目配置：项目名、`Visible Meta Files` / `Force Text` 序列化、确认 URP 渲染管线
- [ ] 1.3 新版 Input System 接入 + 基础 Action Map（Move / Look / Jump），为 M2 动态重绑留结构
- [ ] 1.4 角色控制器：移动 + 跳跃 + 重力（CharacterController 方案）+ 第三人称跟随相机
- [ ] 1.5 白盒测试房间：地板、围墙、一个可跳的台阶/小障碍
- [ ] 1.6（可选）GameManager / 场景管理最小框架，为 M2 关卡切换留接口（可推迟到 M2.8）

## 备注
- **视角默认第三人称跟随相机**；若团队倾向第一人称或俯视 2.5D，尽早提出调整（影响 M2 推动箱子的可视性）。
- **角色用 CharacterController**（解谜 + 物理交互更可控）；若 M2 推动箱子需要真实物理再评估是否切 Rigidbody。
- M1 的移动/跳跃是硬编码直连；M2.1 会把它重构包装成「能力(Ability)」，此处不提前抽象。
- Unity 工程文件不要手工编辑 meta，让 Editor 生成。
