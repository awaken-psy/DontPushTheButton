# 微调参数表 (Tuning Parameters)

> **单一事实来源**：记录所有「后期可微调」的参数（手感 / 数值 / 阈值），供调参与 playtest 对照。
>
> **维护规则**：每完成一个 milestone 小节（`M{N}.x`），若新增或修改了可调参数，**必须同步更新本表**（见 `.claude/rules/milestone-rules.md`）。参数化规范见 `.claude/rules/code-conventions.md`。
>
> **调参方式速查**：
> - **SO 字段** → 改 `.asset`（Play 中即时生效，重启 Play 更稳）
> - **组件 `[SerializeField]`** → 改 Inspector
> - **场景对象属性**（如 `CM TopDown`）→ 改场景 Inspector
> - **editor builder const** → 改 const + 重跑菜单（一般无需，运行时调场景对象即可）

## M1 · 工程骨架与核心移动

### 移动 / 跳跃 — `MovementTuning` SO
> 资产：`Assets/Config/MovementTuning.asset` · 来源：M1.4

| 字段 | 当前值 | 单位 | 调什么手感 |
|------|--------|------|-----------|
| `_moveSpeed` | 5 | m/s | 水平移动快慢 |
| `_jumpHeight` | 1.5 | m | 跳跃高度（反推起跳速度） |
| `_gravity` | -20 | m/s² | 下落快慢（负 = 向下） |
| `_turnSpeed` | 720 | °/s | 转向速度（`<=0` = 瞬切朝向） |
| `_groundStickVelocity` | -2 | m/s | 贴地钳制（grounded 时下落速度上限，避免无限累积） |

### PlayerMover 组件 `[SerializeField]`
> 挂在：Player · 来源：M1.4

| 字段 | 当前值 | 说明 |
|------|--------|------|
| `_moveInputThreshold` | 0.01 | 输入死区：水平输入向量平方长度低于此值视为静止（不移动/不转向） |
| `_tuning` | MovementTuning.asset | 数值引用 |
| `_relativeCamera` | Main Camera | 水平移动方向所相对的相机 |

### 落点标记 — `LandingMarker` 组件
> 挂在：LandingMarker（黄色圆盘） · 来源：M1.4

| 字段 | 当前值 | 说明 |
|------|--------|------|
| `_groundY` | 0.02 | 圆盘吸附的地面 Y 高度 |
| `_target` | Player | 跟随目标（角色） |

### 俯视相机 — Cinemachine
> 场景对象：`CM TopDown`（运行时调，最常用） · builder：`Assets/Editor/CinemachineTopDownBuilder.cs`（const，改初始默认值需重跑菜单 `DPTB/Build Top-Down Camera`） · 来源：M1.4

| 参数 | 当前值 | 位置 | 调什么 |
|------|--------|------|--------|
| 俯角 Pitch | 50° | CM TopDown Transform.rotation.x / builder const | 相机俯视角度 |
| 偏航 Yaw | 0° | 同上 | 相机水平偏转 |
| 距离 Distance | 14 | FramingTransposer.m_CameraDistance / builder const | 相机距角色远近 |
| 死区 W/H/Depth | 0 | FramingTransposer.m_DeadZoneWidth/Height/Depth | 跟随死区（0 = 硬跟随） |
| 阻尼 X/Y/Z | 0 | FramingTransposer.m_XDamping 等 | 跟随平滑度（0 = 无） |

### CharacterController（Unity 内置，Inspector 调）
> 挂在：Player · 来源：M1.4

| 属性 | 当前值 | 调什么 |
|------|--------|--------|
| radius | 0.5 | 碰撞胶囊半径 |
| height | 2 | 胶囊高度 |
| stepOffset | 0.3 | 自动踏上低台阶的高度（< 此值可直接走上去） |
| slopeLimit | 45° | 可行走坡度 |

## M3.10 · 能力状态接口（给代码控制 transform 的动画读）

> 主角是机器人，队友用**代码控制部件 Transform** 做程序动画（不用 Animator）。本节登记暴露给动画读取的状态接口（只读信号，非可调数值）。详细交接见 `docs/design/动画系统.md`。

### 各能力/控制器暴露给动画的状态接口
| 接口 | 位置 | 用途（动画怎么用） |
|------|------|------|
| `JumpAbility.IsFlying` | JumpAbility.cs | 超载飞行中（如螺旋桨加速） |
| `JumpAbility.IsSlowFalling` | JumpAbility.cs | 缓落态 |
| `PickupAbility.IsCarrying` | PickupAbility.cs | 搬运中（手臂/枪姿态） |
| `PickupAbility.OnCarryChanged` 事件 | PickupAbility.cs | 捡起(true)/放下(false)瞬间——切搬运姿态 |
| `PlayerAbilityController.MoveInput` | PlayerAbilityController.cs | 移动方向+幅值（轮子转速/车身倾斜） |
| `PlayerAbilityController.IsGrounded` | PlayerAbilityController.cs | 地面/空中 |
| `PlayerAbilityController.VerticalVelocity` | PlayerAbilityController.cs | 上升/下落（车身俯仰） |
| `PlayerAbilityController.OnInstantCast` 事件 | PlayerAbilityController.cs | Dash/Pull/Push 触发瞬间（车身前倾等） |

> 这些是只读状态/事件，非可调数值。动画侧的转速/角度参数走队友自己脚本的 `[SerializeField]`，由队友登记。
