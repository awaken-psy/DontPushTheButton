# 微调参数表 (Tuning Parameters)

> **单一事实来源**：记录所有「后期可微调」的参数（手感 / 数值 / 阈值），供调参与 playtest 对照。
>
> **维护规则**：每完成一个 milestone 小节（`M{N}.x`），若新增或修改了可调参数，**必须同步更新本表**（见 `.claude/rules/milestone-rules.md`）。参数化规范见 `.claude/rules/code-conventions.md`。
>
> **调参方式速查**：
> - **SO 字段** → 改 `.asset`（Play 中即时生效，重启 Play 更稳）
> - **组件 `[SerializeField]`** → 改 Inspector
> - **场景对象属性**（如 `CM TopDown`）→ 改场景 Inspector
> - **editor builder const** → 改 const + 重跑菜单（一般无需，运行时调场景对象即可）

## M1 · 工程骨架与核心移动

### 移动 / 跳跃 — `MovementTuning` SO
> 资产：`Assets/Config/MovementTuning.asset` · 来源：M1.4

| 字段 | 当前值 | 单位 | 调什么手感 |
|------|--------|------|-----------|
| `_moveSpeed` | 5 | m/s | 水平移动快慢 |
| `_jumpHeight` | 1.5 | m | 跳跃高度（反推起跳速度） |
| `_gravity` | -20 | m/s² | 下落快慢（负 = 向下） |
| `_turnSpeed` | 720 | °/s | 转向速度（`<=0` = 瞬切朝向） |
| `_groundStickVelocity` | -2 | m/s | 贴地钳制（grounded 时下落速度上限，避免无限累积） |

### PlayerMover 组件 `[SerializeField]`
> 挂在：Player · 来源：M1.4

| 字段 | 当前值 | 说明 |
|------|--------|------|
| `_moveInputThreshold` | 0.01 | 输入死区：水平输入向量平方长度低于此值视为静止（不移动/不转向） |
| `_tuning` | MovementTuning.asset | 数值引用 |
| `_relativeCamera` | Main Camera | 水平移动方向所相对的相机 |

### 落点标记 — `LandingMarker` 组件
> 挂在：LandingMarker（黄色圆盘） · 来源：M1.4

| 字段 | 当前值 | 说明 |
|------|--------|------|
| `_groundY` | 0.02 | 圆盘吸附的地面 Y 高度 |
| `_target` | Player | 跟随目标（角色） |

### 俯视相机 — Cinemachine
> 场景对象：`CM TopDown`（运行时调，最常用） · builder：`Assets/Editor/CinemachineTopDownBuilder.cs`（const，改初始默认值需重跑菜单 `DPTB/Build Top-Down Camera`） · 来源：M1.4

| 参数 | 当前值 | 位置 | 调什么 |
|------|--------|------|--------|
| 俯角 Pitch | 50° | CM TopDown Transform.rotation.x / builder const | 相机俯视角度 |
| 偏航 Yaw | 0° | 同上 | 相机水平偏转 |
| 距离 Distance | 14 | FramingTransposer.m_CameraDistance / builder const | 相机距角色远近 |
| 死区 W/H/Depth | 0 | FramingTransposer.m_DeadZoneWidth/Height/Depth | 跟随死区（0 = 硬跟随） |
| 阻尼 X/Y/Z | 0 | FramingTransposer.m_XDamping 等 | 跟随平滑度（0 = 无） |

### CharacterController（Unity 内置，Inspector 调）
> 挂在：Player · 来源：M1.4

| 属性 | 当前值 | 调什么 |
|------|--------|--------|
| radius | 0.5 | 碰撞胶囊半径 |
| height | 2 | 胶囊高度 |
| stepOffset | 0.3 | 自动踏上低台阶的高度（< 此值可直接走上去） |
| slopeLimit | 45° | 可行走坡度 |

## M2.1 · 能力系统骨架（移动 / 跳跃 / 推动）

### 推动能力 — `PushTuning` SO
> 资产：`Assets/Config/PushTuning.asset` · 来源：M2.1

| 字段 | 当前值 | 单位 | 调什么 |
|------|--------|------|--------|
| `_range` | 2 | m | 推动作用距离（按 E 时前方此距离内命中才推） |
| `_pushDistance` | 1.5 | m | 单次推动位移 |
| `_cooldown` | 0 | s | 推动冷却（`<=0` = 无；M3 调参） |

### 能力组件 `[SerializeField]`
> 挂在：Player · 来源：M2.1

| 组件 | 字段 | 当前值 | 说明 |
|------|------|--------|------|
| MoveAbility | `_moveInputThreshold` | 0.01 | 输入死区（从 PlayerMover 迁移，值不变） |
| PushAbility | `_pushTuning` | PushTuning.asset | 推动调参引用 |
| PushAbility | `_rayHeight` | 0.5 | m | 射线发射世界 Y 高度（对准物体中心；1m 箱子中心约 0.5） |
| PlayerAbilityController | `_tuning` | MovementTuning.asset | 运动调参引用（从 PlayerMover 迁移） |
| PlayerAbilityController | `_relativeCamera` | Main Camera | 水平移动相对相机（从 PlayerMover 迁移） |

> 注：`PlayerMover` 已于 M2.1 退役（移动/跳跃/重力逻辑迁入 `PlayerAbilityController` + `MoveAbility`/`JumpAbility`，新增 `PushAbility`）。M1 段 PlayerMover 参数见上方，仅供历史对照。

## M2.7 · 示范关卡 关2裂口（布局数值）

> 白盒关2布局（SampleScene）。playtest 调（M2.9）。

### 关2几何
| 对象 | 位置(pos) | 尺寸(scale) | 说明 |
|------|----------|------------|------|
| FrontFloor | (0,-0.5,-5.25) | (18,1,6) | 起点平台（顶面 y=0，z=-8.25~-2.25） |
| BackFloor | (0,-0.5,5.25) | (18,1,6) | 对岸平台（z=2.25~8.25） |
| Pit | (0,-8,0) | (18,1,5) | 沟底（深，FallReset y<-5 先触发） |
| 沟（空隙） | z=-2.25~2.25 | — | **宽 4.5m**（普通跳 3.87 过不去） |
| TestCrate | (0,0.5,-3.5) | (1,1,1) | 关2箱子（Pushable），推到 z=0 做中部踏板 |
| LevelExit | (0,1,6.5) | (4,2,1) | 出口触发器（绿色 cube，tag Player 触发） |

### FallReset（Player 上）
| 字段 | 值 | 说明 |
|------|-----|------|
| `_fallThreshold` | -5 | 掉沟 y 阈值 |
| `_respawnPos` | (0,1,-5) | 重置起点 |
| `_respawnEuler` | (0,0,0) | 朝 +z（沟方向） |

> 解法：A) 推箱到 z=0 踏板 → 两段跳每段 2.25m（<3.87）；B) 超载跳 ≈4.85m（>4.5）。
