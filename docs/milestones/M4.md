## 任务详述

### 4.1 Demo 完整性检查 ✅
- **做了什么**：以「玩法验证 demo」视角对 `LevelPreview1-3` 走查 6 维度，产出缺陷清单。完整性 ≠ 商品质素，聚焦：①核心循环 ②关1/2/3 可玩（关4/5 待队友）③双解法 ④评分腐败 ⑤菜单流程（搁置）⑥blocking bug。
- **依据**：玩法验证 demo 定位；GDD `7`
- **分工定位**：4.1 = 开发者视角冒烟（「能不能玩通」），可多次跑；4.9 playtest = 玩家视角乐趣验证（最后做）。
- **验收**：检查表覆盖 6 维度；缺陷清单按优先级排序。✅ 达成

### 4.2 LevelPreview1-3 多关机制 + 系统接入（最高优先）
- **做什么**：让 `LevelPreview1-3` 从「几何预览」变「可玩多关场景」。修 4.1 发现的 B+C 类：
  - **B 检查点切关机制**（新写）：写 `CheckPoint` 脚本（玩家碰触发器 → 通知 GameManager 切当前关 + 更新出生点）；GameManager 加多关状态（CurrentLevel + 按关切 LevelConfig）；当前 3 个检查点 collider **无脚本**不响应
  - **B 过关/切关触发**：关末出口（LevelExit 或末检查点联动）→ 切下一关；末关 → 通关（当前 `_levelExit=null`、无 LevelExit 对象）
  - **C 评分系统接入**：场景放 ScoreSystem/StarRating/ScorePopup（M3.7 代码在，场景缺），接 GameManager 过关事件（当前 `_scorePopup=null`）
  - **C 腐败视效接入**：配 Global Volume 的 VolumeProfile（Vignette + ColorAdjustments），接 CorruptionTracker 层级驱动（M3.8 代码在，当前 `sharedProfile=null`）
- **顺带**：检查点重命名（CheckPoint1/CheckPoint/CheckPoint 重名 → CheckPoint_Lv1/2/3）
- **依据**：4.1 缺陷清单 B/C 类
- **为什么最高优先**：demo 可玩的前提；没它 4.3/4.4 优化无的放矢
- **验收**：`LevelPreview1-3` 跑通 配置→玩关1→检查点切关2→…→过关→评分弹窗→腐败视效反馈 全闭环
- **依赖**：M3.6 / M3.7 / M3.8（代码在）；4.1 ✅
- **测试方式**：`MCP 验证` + `手动验证`
