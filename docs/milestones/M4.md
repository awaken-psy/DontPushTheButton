## 任务详述

### 4.1 Demo 完整性检查（最优先）
- **做什么**：以「玩法验证 demo」视角对全工程做完整性走查，产出 **检查表 + 缺陷清单**；后续 4.2/4.3/4.6 从缺陷清单认领优先级。完整性 ≠ 商品质素，聚焦 6 个维度：①核心循环能否跑通（配置绑键 → 进关 → 普通/超载过关 → 腐败满格重置）；②5 关是否都可玩可通关；③双解法关两条路线是否都成立；④评分（4 星）/ 腐败层级是否正常工作；⑤主菜单 → 进关 → 结算流程是否串通（关卡选择已移除，走线性推进）；⑥有无 blocking bug / 引用断 / 资源缺。
- **依据**：玩法验证 demo 定位；GDD `7`
- **为什么最优先**：它是后续所有优化任务的「需求源」——先查清问题在哪，再决定手感/腐败/数值各改什么
- **验收**：检查表覆盖 6 维度；产出按优先级排序的缺陷清单；blocking bug 已修或转 `[!]`
- **依赖**：M3 完成
- **测试方式**：`手动验证`（走查）+ `MCP 验证`（read_console 查报错 / find_gameobjects 查引用断）
- **分工定位**：4.1 = 开发者视角冒烟（「能不能玩通」），可多次重复跑；4.8 playtest = 玩家视角乐趣验证（「好不好玩」），最后一次性做。两者不重复。

### 4.2 能力手感优化（操作反馈层）
- **做什么**：优化各核心能力（推动 / 拉动 / 冲刺 / 跳跃飞行）的**操作反馈**——输入响应延迟、起跳/落地前摇后摇、推箱拉物的力度曲线与停顿、dash 起停与镜头、飞行高度维持手感、能力触发时动画/音效/镜头反馈的衔接。
- **维度边界**：只管「感觉」（响应、衔接、反馈），**不动「平衡数」**（距离/倍率/冷却归 4.6）。若为改善手感临时调了数值，先记下来留给 4.6 统一收口。
- **依据**：GDD `3`（能力）；4.1 缺陷清单
- **验收**：核心能力手感自评合格 + 4.1 缺陷清单对应项关闭
- **依赖**：M3.6（能力实现）+ 4.1（认领缺陷）
- **测试方式**：`手动验证`（手感观察）+ `MCP 验证`

### 4.3 腐败系统优化（系统博弈层）
- **做什么**：优化腐败 5 层级的**系统博弈体验**——各层惩罚的触发节奏（第几层开始降速/glitch/冷却延长）、与超载模块的博弈张力（超载收益 vs 腐败代价是否有趣）、层级间视效与反馈的递进。**合并原 4.1「腐败层级微调」**。
- **维度边界**：管「系统体验与节奏」；具体惩罚数值的最终标定归 4.6。本轮先按设计直觉 + 4.1 缺陷调到「感受上对」，最终校准留 4.8 playtest。
- **依据**：GDD `4.4` 腐败层级；M3.8（腐败实现）；4.1 缺陷清单
- **验收**：5 层级惩罚递进手感自评合理；超载博弈有张力
- **依赖**：M3.8 + 4.1
- **测试方式**：`手动验证` + `MCP 验证`

### 4.4 配置阶段 3D 实时预览
- **做什么**：配置阶段的关卡预览从「无/静态截图」升级为 **3D 实时预览 + 相机轨道控制**，玩家可旋转查看障碍布局。
- **依据**：GDD `4.1`（关卡全貌预览）；`toConfirm §J6`
- **验收**：配置时能 3D 旋转查看关卡
- **依赖**：M2.4
- **测试方式**：`手动验证` + `MCP 验证`
- **备注**：已有进行中工作——LevelPreview 场景搭了 CinemachineBrain + TopDown 相机 + FollowOffset(0,8,-6)，卡在「相机 yaw 不跟随玩家朝向」（见 [[unity-cinemachine-2.5d-camera]]）。本任务承接该工作。

### 4.6 数值精调（平衡收口层）
- **做什么**：全局数值**统一平衡收口**——腐败增量、强化倍率、跳跃距离、飞行时长、节制阈值、dash 语义、各能力冷却。汇总 4.2/4.3 为手感临时调的数值，统一标定到平衡自洽。
- **维度边界**：管「平衡数」，是 4.2/4.3 之后的收口，确保整体数值自洽（某能力加强后腐败代价是否对等）。
- **依据**：GDD `7`；`toConfirm §H`；4.2/4.3 累积的数值改动
- **验收**：数值全局自洽、手感到位
- **依赖**：4.2 + 4.3 大致完成
- **测试方式**：`手动验证` + `MCP 验证`；最终校准留 4.8

### 4.7 结局（解谜化 + 动画）
- **做什么**：关 5 终点——玩家把「发送信号」能力绑到某键按下触发结局（键位配置主题收尾），然后通讯阵列点亮 + 信号射向星空动画。
- **依据**：GDD `2.5`（结局解谜化）；v1.0 整合（吸收队友）
- **验收**：结局按键解谜 + 动画都实现
- **依赖**：M3.6（关 5 跑通）；可与 4.6 并行
- **测试方式**：`手动验证` + `MCP 验证`

### 4.8 全 5 关 playtest（最终验证）

- **验收**：全 5 关 playtest 记录；双解法关两条路线都有人走
