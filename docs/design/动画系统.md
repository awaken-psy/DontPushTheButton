# 主角动画交接（机器人 · 代码控制 transform）

> 给**动画/美术队友**。主角是轮式机器人（车身 + 轮子 + 螺旋桨 + 枪的部件组合），动画用**代码控制部件 Transform**（程序动画，不用 Mecanim Animator / 动画 clip / Humanoid rig）。
> 本文列出代码里能读的**状态信号**——你写动画代码时读这些，驱动部件旋转/倾斜。
> 接口契约：改接口名要同步能力侧代码（Claude 维护）。关联 M3.10。

---

## 1. 方案（一句话）

不用 Animator。机器人的"动画" = 你写个脚本挂 Player 上，在 `LateUpdate` 里**旋转/摆动部件 Transform**（轮子转、螺旋桨转、车身倾斜）。读下面的状态信号决定怎么转。

---

## 2. 状态接口清单（你读这些）

都挂在 Player 那个 GameObject 上，`GetComponent<T>()` 拿：

| 想做的动画 | 读什么 | 组件 | 类型 | 说明 |
|---|---|---|---|---|
| 轮子转速 / 车身倾斜方向 | `MoveInput` | `PlayerAbilityController` | `Vector2` | 移动输入（-1..1，相对屏幕）；`.magnitude` = 速度 |
| 地面 vs 空中 | `IsGrounded` | `PlayerAbilityController` | `bool` | `CharacterController.isGrounded` |
| 上升 / 下落（车身俯仰） | `VerticalVelocity` | `PlayerAbilityController` | `float` | 正 = 上升，负 = 下落 |
| 超载飞行中（螺旋桨加速） | `IsFlying` | `JumpAbility` | `bool` | 飞行状态机的 Flying 态 |
| 缓落 | `IsSlowFalling` | `JumpAbility` | `bool` | 飞行结束后的缓落 |
| 搬运中（手臂 / 枪姿态） | `IsCarrying` | `PickupAbility` | `bool` | 拾取搬运的 Carrying 态 |
| **捡起 / 放下瞬间**（切搬运姿态） | `OnCarryChanged` | `PickupAbility` | `event Action<bool>` | `true`=捡起、`false`=放下；订阅同 OnInstantCast：`pickup.OnCarryChanged += pickedUp => { ... }` |
| Dash / Pull / Push 触发瞬间 | `OnInstantCast` | `PlayerAbilityController` | `event Action<AbilityKind>` | 能力确认执行时触发，参数 = `Dash`/`Pull`/`Push` |

> 这些接口有契约单测保着（`AbilityStateContractTests`），不会偷偷改坏。

## 3. 代码示例（拿引用 + 订阅事件）

```csharp
using UnityEngine;
using DontPushTheButton.Abilities;
using DontPushTheButton.Player;

public class ProtagonistRigAnimator : MonoBehaviour
{
    [Header("部件引用（Inspector 拖，别用 GetChild 硬编码）")]
    [SerializeField] private Transform _wheels;
    [SerializeField] private Transform _airscrew;
    [SerializeField] private Transform _body;

    [Header("参数（可调）")]
    [SerializeField] private float _wheelDegPerSec = 720f;
    [SerializeField] private float _airscrewDegPerSec = 1800f;

    private PlayerAbilityController _controller;
    private JumpAbility _jump;

    private void Awake()
    {
        _controller = GetComponent<PlayerAbilityController>();
        _jump = GetComponent<JumpAbility>();
    }

    private void OnEnable()
    {
        if (_controller != null) _controller.OnInstantCast += OnCast;
    }

    private void OnDisable()
    {
        if (_controller != null) _controller.OnInstantCast -= OnCast;
    }

    private void OnCast(AbilityKind k)
    {
        if (k == AbilityKind.Dash) { /* 车身前倾一次 */ }
    }

    private void LateUpdate()
    {
        if (_controller == null) return;
        float speed = _controller.MoveInput.magnitude;
        bool flying = _jump != null && _jump.IsFlying;

        if (_wheels != null)
            _wheels.Rotate(Vector3.right, _wheelDegPerSec * speed * Time.deltaTime, Space.Self);
        if (_airscrew != null)
            _airscrew.Rotate(Vector3.up, _airscrewDegPerSec * (flying ? 2f : 1f) * Time.deltaTime, Space.Self);
    }
}
```

## 4. prefab 前提（Player 根要挂的脚本）

机器人拼成 Player prefab 时，根 GameObject 上挂这些（Claude 写的能力脚本，让机器人能用能力）：
- `CharacterController`（Unity 内置，碰撞移动）
- `PlayerAbilityController`（物理/输入权威，**必须**）
- `MoveAbility` / `JumpAbility`（移动/跳跃）
- `PickupAbility` **或** `PushAbility`（搬运 vs 光束推，二选一，关卡定）
- `PullAbility` / `DashAbility`（拉/冲刺）
- `CorruptionTracker`（腐败）
- 你自己的动画脚本（如上面的 `ProtagonistRigAnimator`）

各能力的 tuning SO 引用要在 Inspector 连好（`PickupTuning.asset` / `PushTuning.asset` / ...），否则能力 Tick 第一行 `if (_tuning == null) return` 静默不执行。

部件（轮子/螺旋桨/车身/枪）作为 Player 的**子物体**，你脚本里 `[SerializeField] private Transform _wheels;` 拖引用。

## 5. 模型导入（fbx Rig）

机器人 SM_ 部件 fbx（`Assets/Arts/Protagonist/Models/`）：
- 纯代码转部件 → Rig 选 **None**（不进动画系统，最干净）
- 若也想用 Animator 放点 clip → **Generic**
- ⚠️ **别用 Humanoid**——机器人非人形，Unity 的人形骨骼映射对它无意义（当前 fbx 是 Humanoid，要改）

---

## 6. 动画建议（参考，具体你定）

- **轮子**：`MoveInput.magnitude` 越大转越快
- **螺旋桨**：常驻转，`IsFlying` 时加速
- **车身**：移动后仰、`VerticalVelocity > 0` 上仰、`OnCast(Dash)` 前倾
- **落地**：`IsGrounded` 从 false→true 那帧压扁一下
- 平滑用 `Quaternion.Slerp`；所有转速/角度走 `[SerializeField]` 可调

---

## 7. 约定（别踩坑）

- 朝向 + 位移由 `PlayerAbilityController` + `MoveAbility` 代码驱动。你的动画**只改部件的局部 Transform**（`_wheels.localRotation` 等），**别动 Player 根的 position/rotation**——会和物理冲突。
- 调参（转速/倾斜角）走你脚本的 `[SerializeField]`，并登记到 `docs/config/微调参数表.md`。
- 拿部件引用用 `[SerializeField]` 拖，**不要 `transform.GetChild(N)` 硬编码**（子物体顺序一变就断）。
