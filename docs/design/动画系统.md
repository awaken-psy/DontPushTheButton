# 动画系统交接（Player 全身人形动画）

> 给**动画/美术队友**的接入说明。代码侧已把能力/运动状态桥接到一组 Animator 参数，你只需按本文建 Animator Controller + 导入人形 rig，对着参数表做状态机即可，**不用改任何代码**。
> 参数表是双方的契约——**改参数名必须同步** `Assets/Scripts/Player/PlayerAnimator.cs`（Claude 维护）和你的 Controller（你维护）。
> 关联里程碑：M3.10（动画驱动接口）。能力机制 SoT 见 `GDD_键位重构_v1.0.md §4.2`。

---

## 1. 现状（你接手前是什么样）

- **玩家现在是个胶囊**：`SampleScene.unity` 里名为 `Player` 的 GameObject，用 Unity 内置 Capsule mesh + `MeshRenderer`，**没有任何 Animator / SkinnedMeshRenderer / Avatar / 骨骼**。整个项目零动画资产。
- **运动/能力全由代码驱动**：`PlayerAbilityController`（物理权威）+ 一组 `Ability` 组件（Move/Jump/Pickup/Pull/Dash/Push）。玩家朝向、位移、跳跃、冲刺全是代码算的，动画只是"表现层"叠加。
- **CharacterController**：center=(0,0,0)，height=2，radius=0.5 → **足底在本地 y = -1.0**（Transform 原点下方 1m）。

---

## 2. 你要做的事（接入步骤）

### 2.1 导入人形模型 + Avatar
1. 把人形角色 fbx 放进 `Assets/Models/Characters/`。
2. fbx Import Settings → **Rig → Animation Type = Humanoid** → Apply（Unity 自动生成 Avatar）。
3. 角色 mesh 的**脚底要对齐 CharacterController 底部**（本地 y=-1.0）：
   - 把模型作为 `Player` 的**子物体**，调子物体 Local Position 的 y 使脚底落在 -1.0；或
   - 在 fbx Import 的 Transform 里调 offset。
   - （千万别为了对齐去动 `Player` 自身的 Transform——它是 CharacterController 的物理原点。）

### 2.2 加 Animator 组件
1. 在 `Player` GameObject 上加 `Animator` 组件，Controller 指向你的 `Player.controller`，Avatar 指向 2.1 生成的。
2. **关掉 `Apply Root Motion`**（关键！）——玩家朝向和位移已由 `MoveAbility` / `PlayerAbilityController` 代码强制驱动，开了 Root Motion 会和代码打架（角色乱飘/朝向错乱）。Animator 用 in-place 动画。

### 2.3 挂 PlayerAnimator 脚本
1. 在 `Player` 上加 `PlayerAnimator` 组件（`Assets/Scripts/Player/PlayerAnimator.cs`）。
2. 它会自动 `GetComponent` 取 Animator / PlayerAbilityController / JumpAbility / PickupAbility / CorruptionTracker，**无需拖引用**。
3. 它在 `LateUpdate` 每帧把状态写进 Animator 参数；瞬时能力（冲刺/拉/推）经事件 `SetTrigger`。

> **容错**：`PlayerAnimator` 只 set Controller 里**已定义**的参数，未定义的静默跳过（不刷 warning）。所以你可以**增量建 Controller**——先建 `Speed`/`IsGrounded` 跑通走/待机，再逐个加其余参数，不会报错。

### 2.4 资产目录约定
- 模型：`Assets/Models/Characters/`
- Animator Controller：`Assets/Animation/Controllers/Player.controller`
- Animation Clips：`Assets/Animation/Clips/`
- （目录没有就建）

## 3. Animator 参数契约（核心 —— 你照此建 Parameter）

| Parameter | 类型 | 来源（代码） | 值域 / 说明 |
|---|---|---|---|
| `Speed` | Float | `PlayerAbilityController.MoveInput.magnitude` | 0..1，walk/run blend tree 驱动 |
| `MoveX` | Float | `MoveInput.x` | 可选，strafe blend（俯视视角可只用 Speed + 代码朝向） |
| `MoveY` | Float | `MoveInput.y` | 可选，同上 |
| `VerticalVelocity` | Float | `PlayerAbilityController.VerticalVelocity` | 正=上升，负=下落；判 jump fall |
| `IsGrounded` | Bool | `PlayerAbilityController.IsGrounded` | 地面/空中主切换 |
| `IsFlying` | Bool | `JumpAbility.IsFlying` | **超载飞行**中（持续上升期） |
| `IsSlowFall` | Bool | `JumpAbility.IsSlowFalling` | 超载飞行结束后的缓落 |
| `IsCarrying` | Bool | `PickupAbility.IsCarrying` | 正在搬运物体 |
| `Corruption` | Float | `CorruptionTracker.Normalized` | 0..1，可做泛红/过载视效层 |
| `Dash` | Trigger | `OnInstantCast(Dash)` | 冲刺**确认执行**瞬间 |
| `Pull` | Trigger | `OnInstantCast(Pull)` | 拉动/抓钩确认执行瞬间 |
| `Push` | Trigger | `OnInstantCast(Push)` | 推光束命中并推动瞬间 |

**命名规则**：PascalCase；Bool 用 `Is` 前缀；Trigger 名与 `AbilityKind` 枚举同名。改任何名字都要同步 `PlayerAnimator.cs` 顶部那组 `const string`。

**建议的优先级**（先做能跑起来的最小集）：
1. `Speed` + `IsGrounded` → 待机/走/跑/跳 blend（先有这个角色就"活了"）
2. `Dash` Trigger → 冲刺表现
3. `IsCarrying` → 搬运姿态
4. 其余（飞行/缓落/拉推/腐败）按需加

## 4. 状态来自代码的哪里（查代码用）

| 状态 | 代码位置 |
|---|---|
| 移动/朝向 | `Assets/Scripts/Abilities/MoveAbility.cs`（朝向 `Body.rotation` 在 :40-46，朝移动方向） |
| 飞行/缓落状态机 | `JumpAbility.cs`，`FlyState { Ground, Ascend, Flying, SlowFall }`（内部枚举，对外暴露 `IsFlying`/`IsSlowFalling`） |
| 搬运状态机 | `PickupAbility.cs`，`CarryState { Idle, Carrying }`（对外 `IsCarrying`） |
| 瞬时能力事件出口 | `IAbilityContext.NotifyCast(AbilityKind)` ← `PlayerAbilityController.NotifyCast` → raise `OnInstantCast` 事件（Dash/Pull/PushAbility 在确认执行点调用） |
| 腐败值 | `CorruptionTracker.Normalized`（0..1）+ `OnChanged` 事件 |
| 物理/输入权威 | `PlayerAbilityController.cs`（`IAbilityContext` 实现，每帧 `Update` 流水线） |

> **朝向提醒**：玩家朝向**只跟移动方向**（不是相机方向），`TurnSpeed=720°/s` 近瞬时但有平滑。转向动画若想要 pivot/turn blend，可用 `MoveInput` 变化率自己估，或后续让代码额外暴露角速度（目前未暴露）。

## 5. 协作分工

| 谁 | 负责什么 |
|---|---|
| **动画/美术（你）** | 模型导入 + Avatar + Animator Controller + Animation Clips + 视觉调参（过渡时长、blend tree 阈值都在 Controller 里，**不进代码**） |
| **代码（Claude）** | `PlayerAnimator.cs` 参数出口 + 各能力暴露的状态属性 + `OnInstantCast` 事件。**改参数名/加新参数找代码这边同步** |

**改参数名流程**：任一方要改 → 在本文档 + `PlayerAnimator.cs` const + Controller 三处同步 → 提交。

---

## 6. ⚠️ 注意事项

1. **`Apply Root Motion` 必须关**（见 2.2）。
2. **`SampleScene` 里 `PlayerAbilityController` 当前是禁用状态**（`m_Enabled: 0`，疑似测试场景残留）。接入动画前先在 Inspector 确认它**启用**，否则玩家不会动、`Speed`/`IsGrounded` 不刷新（PlayerAnimator 不依赖它启动，但状态值不更新）。正式关卡场景以 `Level_01` 为准。
3. **`PlayerAnimator` 是纯表现层**，不参与物理/输入判定——删掉它角色照常能玩，只是没动画。调试物理 bug 时可以先禁用 PlayerAnimator 排除干扰。
4. **无可调数值参数**：`PlayerAnimator` 不含任何 `[SerializeField]` 数值字段；所有"速度到多少切跑""过渡多久"都在 Animator Controller 里调。符合本项目"调手感不改代码"约定（见 `.claude/rules/code-conventions.md`）。
5. **满格腐败重置**（`OnCorruptionFull`）目前未映射成 Trigger，如需"满格爆发/重置"动画，找代码侧加一个 `Reset` trigger。
