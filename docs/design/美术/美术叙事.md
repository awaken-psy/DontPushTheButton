# 故事背景

由于人类不珍惜和平，肆意妄为地挑起战争，其规模不断扩大，最终爆发核战争（或者大规模战争），人类从此在地球上销声匿迹。而主角就是战争末期研究的特种兵器原型，但随着核战爆发研究所被无差别打击，主角因此得到了被意外启动的机会。在人类灭绝（可以在场景中暗示还有人幸存）不知多久之后，主角被意外启动并从破败的研究所中醒来，由于尚处于试验阶段，主角在没有外部命令控制的情况下可以自由行动，由此进入游戏开场。

*背景意欲着人类没有做到“别按那个按钮”，按下了战争这个“按钮”，结果导致人类自食其果遭受灭顶之灾。*



# 场景风格

整体为末日废土风，主要场景包含破败的工业场所、千疮百孔的郊区、废墟一般的城市

## 研究所

前期场所，包含`科研、军事、工业`等元素，主要参考对象为*《传送门1\&2》*中的光圈实验室。

## 郊区

中期场所，包含`战场、自然、尸骸`等元素，主要表现大自然正在从战火中慢慢恢复，但往日的人类不见踪影，烘托萧瑟的氛围

## 城市

后期场所，包含`废墟、武器、载具`等元素，主要表现战争对人类文明的摧残，以及人类虽能做到灭亡了自己，但自然却仍能慢慢恢复并侵蚀人类造物的意境

# 主角目的

结合“别按那个按钮”这个主题，主角的目的可以是：

1. 设计师为主角设置了底层代码，如果长时间未收到命令则前往城市中己方的军事设施按下核弹按钮，即使事实上核战早已爆发，但底层代码仍让主角前往。最终主角到达地点并按下按钮，结果什么也没有发生，因为早已成事实

2. 主角的底层代码是杀死所有未标记为己方人员的人类，包括平民。它在郊区游荡时发现了人类的踪迹，并一路追踪到城市，结果发现是幸存的一群平民，此时由场景元素决定玩家选择：场景中存在明显的机关，可以轻易杀死这些幸存者，但玩家可以花费一点脑筋利用场景杀死自己，做到“别按下那个按钮（杀戮）”





你是一台小型勘探无人机，代号 "遥测\-7"（Telemetry\-7），在执行一次外星球表面勘测任务时坠毁。你的母舰残骸散落在这片荒凉星球上，而星球深处某种未知信号源正在持续广播一种高频脉冲，这个脉冲破坏了你的行为逻辑协议。

为了生存并逃离星球，你不得不绕过受损的出厂协议，临时重写自己的运动控制接口——把移动、跳跃、推动、拉动等功能模块重新绑定到不同的输入键上。

然而，某些输入接口已经被外星信号污染成了超载接口。把能力绑定到超载接口上，确实能让你短暂超载、发挥出更强性能；但每次按下，污染信号就会深度写入你的系统，导致视觉干扰、运动误差，最终系统崩溃。
