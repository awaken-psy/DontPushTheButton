# 测试关卡设计标准

## 目的

制定标准并验证玩法，确保核心体验可落地。

颜色图例

- 红色：可推可压的物体

- 橙色：平台

- 深绿色：场景交互物体

- 浅绿色：可把自己拉过去的锚点

- 蓝色：可拉动的物体

## 测试区域

- Jump 区域

- Push 区域

- Pull（拉动）区域

---

## Jump 区域标准

### 设计数值 vs 实际能力

- 水平跳跃距离：关卡设计值 2m / 角色实际能力 2\.2m（10% 余量）

- 垂直跳跃高度：关卡设计值 1m / 角色实际能力 1\.1m（10% 余量）

- 超载状态水平跳跃距离：关卡设计值4m/角色实际能力4\.3m

- 超载状态垂直跳跃高度：关卡设计值2m/角色实际能力2\.2m

### 设计原则

- 关卡按「设计值」搭建，角色能力比设计值多出约 10% 余量

- 余量让玩家「刚好够得着」时有安全缓冲，失败就是真的没跳够

- 余量不超过 15%，避免「跳得太飘」失去重量感

- 不依赖 Coyote Time / Jump Buffer 等输入缓冲系统，通过能力余量本身提供容忍度

## Dash 区域标准

设计数值 vs 实际能力

- 水平 Dash 距离：关卡设计值 3m / 角色实际能力 3\.3m（10% 余量）

- 跳跃 \+ 空中 Dash 组合：关卡设计值 5m / 角色实际能力 5\.5m（10% 余量）

设计原则

- Dash 是纯水平位移，不改变垂直高度

- 垂直爬升只能靠跳跃（1m），Dash 不能用来爬高

- 余量与 Jump 保持一致（10%），操作手感统一

要点

- 纯跳跃：2m

- 纯 Dash：3m（地面能 Dash）

- 跳跃 \+ Dash：5m

## Push 区域标准

### 推动分级

推动速度建议

- 常规推动：角色正常移动速度的 60\-70%（约 2\.5\-3\.5 m/s）

- 超载推动（推普通模块）：角色正常移动速度（约 4\-5 m/s）

- 超载推动（推大模块）：约 1\.5\-2 m/s（体现「费力但位移大」）

- **常规推动**：推普通路模块，每次位移小，需要多次推动

- **超载推动**：

    - 推普通路模块：速度更快

    - 推大模块：只有超载能推动，单次位移大，搭路效率更高

### 设计意图

- 常规推动是基础解法，能通关但费时

- 超载推动是「代价换效率」——增加腐败值，但用大模块快速搭好路

- 玩家权衡：要不要为了效率承担腐败值增加的风险

## Pull 区域标准

### 核心能力

拉力（通过准星OR视线指向的目标类型区分）

### 分级

- **常规 Pull**：拉物体过来（瞄准可拉物体）

- **超载 Pull**：

    - 拉物体更快/更重

    - 可以拉自己过去（瞄准锚点）

### 操作逻辑

- 按键行为统一为「拉」

- 拉什么取决于准星指向的目标类型

- 可拉物体和锚点用不同颜色/标记区分（视觉提示）

### 设计意图

- 超载是「拉力升级」，功能更全但增加腐败值

- 玩家权衡：要不要为了更多可能性承担腐败值增加的风险
