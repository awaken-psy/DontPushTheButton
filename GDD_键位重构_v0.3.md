# Game Design Document: 键位重构 (Key Remap)

**版本**: 0.3  
**日期**: 2026-06-16  
**目标平台**: PC (键盘鼠标)  
**引擎建议**: Unity / Godot / Unreal（按团队熟悉度选择）  
**美术风格**: Lowpoly 3D  
**预计开发周期**: 21 天 Game Jam  

---

## 1. 设计支柱 (Design Pillars)

1. **Meta 即机制**  
   控制面板（键位绑定）不是设置界面，而是核心玩法。解谜的动词是“把能力放到哪个键上”。

2. **限制催生创造力**  
   玩家只有有限数量的“键位槽位”和可用按键。每个选择都意味着放弃另一种选择。

3. **魔键的诱惑**  
   每关都有被“魔化”的按键。把能力绑在魔键上会变强，但按下会触发腐败。玩家自己决定要不要把能力放上去。

4. **Lowpoly 清晰反馈**  
   场景、能力、魔键、腐败效果都用高对比度、几何化的方式呈现，玩家一眼能读出当前风险。

---

## 2. 核心概念 (Core Concept)

你被困在一个由 Lowpoly 几何房间组成的设施中。每个房间都是一个谜题。

进入关卡前，玩家会看到一个**关卡全貌**，然后打开**控制面板**：把能力（移动、跳跃、推动等）绑定到键盘按键上。随着进度推进，可用按键会从最初的 WASD 逐渐扩展。

每关有 1–2 个**魔键**（Demon Key）。把能力绑定到魔键上，该能力会获得**强化效果**；但游戏过程中按下魔键，会触发**腐败**。玩家必须自己决定：
- 是否把核心能力放到魔键上？
- 是否同时保留一个“安全版”能力，占用额外槽位？
- 还是完全避开魔键，用更慢但更稳的方式过关？

**核心体验**: 玩家在“变强但危险”与“安全但受限”之间反复权衡，把键位配置当成一种战前策略。

---

## 3. 核心玩法循环 (Core Gameplay Loop)

### Moment-to-Moment（战斗中）
- **Action**: 玩家按下已绑定能力的按键，与场景互动。
- **Feedback**: 能力生效；如果按下的是魔键，额外触发强化效果 + 腐败累积 + 视听警告。
- **Reward**: 解决眼前障碍或获得捷径。

### Session Loop（单关）
- **Goal**: 在配置阶段规划键位，然后在战斗中抵达出口。
- **Tension**: 魔键提供了更轻松的解法，但每按一次都更接近崩盘。
- **Resolution**: 成功到达出口 → 解锁下一关；腐败满格 → 本关重置，回到配置阶段。

### Long-Term Loop（整局游戏）
- **Progression**: 解锁新能力、新按键、更复杂的房间。
- **Retention Hook**: 追求“零魔键通关”或“最少按键配置通关”的挑战；隐藏房间需要特定键位组合才能进入。

## 4. 机制说明 (Mechanics)

### 4.1 配置阶段 / 控制面板 (Loadout / Control Panel)

**Purpose**: 让玩家在进入关卡前规划键位，是策略核心。  
**Player Fantasy**: “我能在战前决定自己的操作方式。”  
**Input**: 进入关卡前自动打开；鼠标拖拽能力图标到可用按键上；再次点击解除绑定。  
**Output**: 生成该关卡的键位配置；战斗中按下对应按键触发对应能力。  
**Success Condition**: 玩家完成配置并点击“开始关卡”。  
**Failure State**: 玩家未绑定移动能力 → 系统强制提示或重置为默认。  
**Edge Cases**:
  - 玩家把多个能力拖到同一个键 → 后一个覆盖前一个。
  - 玩家把能力拖到魔键上 → 该能力变为强化版，但使用该键会加腐败。
  - 玩家未配置就点开始 → 系统提示“至少绑定移动能力”。
**Tuning Levers**:

  - 默认按键布局 [PLACEHOLDER]
  - 可用按键随关卡解锁的节奏 [PLACEHOLDER]
**Dependencies**: 能力系统、输入系统、魔键系统、UI 系统。

### 4.2 能力系统 (Abilities)

**Purpose**: 提供解谜动词库。  
**Player Fantasy**: “我有多种方式与环境互动。”  
**Input**: 按下已绑定能力的按键。  
**Output**: 触发对应能力效果；若绑定在魔键上，则触发强化版效果。  
**核心能力（Game Jam 最小集）**:

| 能力 | 普通效果 | 魔键强化效果 |
|---|---|---|
| 移动 (Move) | WASD 基础移动 | 移动速度提升 [PLACEHOLDER]% |
| 跳跃 (Jump) | 垂直跃起 | 跳跃高度提升 [PLACEHOLDER]% |
| 推动 (Push) | 对前方物体施加推力 | 范围推力，可同时推动 [PLACEHOLDER] 个物体 |
| 拉动 (Pull) | 把远处物体拉向自己 | 拉动物体的同时拉近玩家 |
| 时间减缓 (Slow) | 短时间内场景速度变慢 | 时间减缓幅度更大、持续更久 |
| 旋转 (Rotate) | 旋转特定机关平台 | 同时旋转多个相邻机关 |

**Tuning Levers**:

  - 作用距离 / 范围 [PLACEHOLDER]
  - 普通与强化效果的倍数 [PLACEHOLDER]
**Dependencies**: 物理系统、输入系统、动画系统。

### 4.3 魔键系统 (Demon Key System)

**Purpose**: 制造核心张力——玩家主动把能力放到魔键上获得强化，但使用时要付出代价。  
**Player Fantasy**: “我能不能承受这把双刃剑？”  
**Input**: 在配置阶段把能力绑定到魔键；在战斗中按下魔键。  
**Output**: 
  - 魔键上的能力触发强化版效果。
  - 腐败值增加，屏幕边缘泛红，音效不和谐。
**Success Condition**: 玩家通过魔键能力完成关卡。  
**Failure State**: 腐败值满格 → 关卡重置，回到配置阶段。  
**魔键规则（Game Jam 版）**:
  - 每关由设计师指定 **1 个魔键**（后期高难度关卡可增至 2 个）。
  - 玩家可以把任意能力绑定到魔键上，但魔键占用一个键位槽位。
  - 同一能力可以同时绑定到普通键和魔键上，消耗 2 个槽位，玩家可按情况选择安全版或强化版。
  - 魔键在不同关卡会变化，例如第 3 关是空格，第 6 关是 E，迫使玩家每关重新规划。
  - 提供无障碍选项：玩家可在系统层把魔键映射到不同的物理按键，但关卡逻辑中的魔键身份不变。

**Edge Cases**:
  - 玩家把所有能力都绑在魔键上 → 允许，但腐败会快速累积，属于高风险配置。
  - 玩家把移动绑在魔键上 → 允许；每移动一次都加腐败，极难通关，但可作为挑战。
  - 玩家不把任何能力绑在魔键上 → 允许；走安全路线，但部分捷径无法使用。
  - 多魔键关卡：每个魔键独立计算腐败增量。

**Tuning Levers**:
  - 每次按下魔键增加的腐败值 [PLACEHOLDER]
  - 腐败上限 [PLACEHOLDER]
  - 腐败是否随时间衰减 [PLACEHOLDER]
  - 每关魔键数量 [PLACEHOLDER]
  - 强化效果倍率 [PLACEHOLDER]
**Dependencies**: 输入系统、腐败系统、UI 反馈系统。

### 4.4 腐败系统 (Corruption System)

**Purpose**: 可视化并量化玩家使用魔键的代价。  
**Player Fantasy**: “世界因为我使用禁忌之力而逐渐崩坏。”  
**Input**: 每次按下魔键增加腐败值。  
**Output**: 根据腐败层级触发不同的视觉、听觉、机制惩罚。  
**腐败层级（Corruption Stack）**:

| 层级 | 腐败值范围 | 效果 |
|---|---|---|
| 0 | 0% | 正常状态 |
| 1 | 1–25% | 屏幕边缘轻微泛红，背景音乐出现不和谐音 |
| 2 | 26–50% | 玩家移动速度略微下降，部分机关行为反转 |
| 3 | 51–75% | 画面出现 glitch 抖动，能力冷却延长 |
| 4 | 76–99% | 重力随机波动，魔键提示声变得急促 |
| 5 | 100% | 世界崩坏，关卡重置 |

**Tuning Levers**:
  - 每层阈值 [PLACEHOLDER]
  - 机关反转概率 [PLACEHOLDER]
  - 重力波动幅度 [PLACEHOLDER]
**Dependencies**: 渲染后处理、音效系统、物理系统。

### 4.5 键位槽位限制 (Key Slot Limit)

**Purpose**: 防止玩家同时带上所有能力，迫使做选择；同时与评分系统挂钩，奖励精简配置。  
**机制**: 
  - 玩家同时只能把能力绑定到 N 个按键上（例如 4 个），其余按键不可用。
  - **吃满槽位本身不导致失败，但会影响评分**。每保留一个空闲槽位，关卡结算时获得额外分数/星级奖励。
  - 空闲槽位奖励鼓励玩家用更少的按键组合过关，而不是把所有能力都带上。
**玩家体验目标**: “我能用更简洁的键位配置过关吗？”  
**Edge Cases**:
  - 玩家把同一能力同时绑定到普通键和魔键 → 占用 2 个槽位，视为已用槽位。
  - 玩家完全不吃满槽位（例如 4 槽中只用 2 个）→ 获得更高的槽位效率奖励。
**Tuning Levers**:
  - 可用槽位数量 [PLACEHOLDER]
  - 是否随进度增加 [PLACEHOLDER]
  - 每空闲一个槽位的评分权重 [PLACEHOLDER]

### 4.6 按键解锁 (Key Unlock)

**Purpose**: 随着进度给予玩家更多操作空间。  
**机制**: 早期关卡只有 WASD 可用；后续关卡解锁 Q、E、Shift、Ctrl、F、R 等按键。  
**Tuning Levers**:

### 4.7 评分与时间系统 (Scoring & Time Pressure)

**Purpose**: 把“配置效率”和“执行速度”转化为可追求的二次目标，给单次通关增加重复可玩性与紧迫感。  
**Player Fantasy**: “我不只要过关，还要用最少的键、最快的速度、最低的腐败完美过关。”  
**机制**:

#### 时间压力
  - 每个关卡有一个**目标时间**（Target Time）和**严格时间**（Hard Time）。
  - 进入关卡后，计时器从 0 开始正向计时，HUD 持续显示当前用时与目标时间。
  - **超时不会导致关卡失败**（避免挫败），但会影响评分。
  - 腐败满格导致关卡重置时，计时器同步重置。

#### 评分维度（星级制，共 4 星）

| 星级 | 条件 | 设计意图 |
|---|---|---|
| ⭐ 通关星 | 抵达出口 | 保证所有玩家都能体验完整内容 |
| ⭐ 时间星 | 在目标时间内完成 | 奖励熟练玩家，制造紧迫感 |
| ⭐ 腐败星 | 结算时腐败值 ≤ 50% | 奖励克制使用魔键的玩家 |
| ⭐ 精简星 | 至少保留 1 个空闲槽位 | 奖励高效键位配置，惩罚无脑吃满 |

#### 分数公式（可选细化）
若需要更细腻的排行榜，可使用以下公式：

```
总分 = 基础通关分(1000)
     + 时间奖励(max 500, 在目标时间内拿满；超时后线性递减至 0)
     + 腐败奖励(max 500, 腐败越低越高)
     + 槽位奖励(每空闲 1 槽 +200, 吃满 = 0)
```

#### HUD 显示
  - 当前用时 / 目标时间
  - 腐败值百分比
  - 已用槽位 / 总槽位
  - 结算界面显示星级与分项得分

**Success Condition**: 玩家通关后看到自己可以优化的三个维度。  
**Failure State**: 无（评分系统是正向激励，不造成硬失败）。  
**Edge Cases**:
  - 玩家用时极短但腐败爆表 → 拿时间星，丢腐败星。
  - 玩家完全不用魔键但用时超长 → 拿腐败星，丢时间星。
  - 玩家吃满槽位但快速低腐败通关 → 丢精简星。
  - 玩家 0 腐败、目标时间内、保留空闲槽位 → 完美 4 星。
**Tuning Levers**:
  - 每关目标时间 [PLACEHOLDER]
  - 每关严格时间 [PLACEHOLDER]
  - 腐败星阈值（建议 50%）[PLACEHOLDER]
  - 精简星所需最少空闲槽位（建议 1）[PLACEHOLDER]
  - 时间奖励递减曲线 [PLACEHOLDER]
**Dependencies**: 计时系统、UI/HUD、结算界面、存档系统。

## 5. 新手引导流程 (Onboarding Flow)

### 关卡 0：走廊
- 目标：走到出口。
- 教学：WASD 移动。
- 配置：强制绑定移动到 W/A/S/D，无魔键。
- 设计要点：无失败可能，30 秒内完成。

### 关卡 1：第一个沟
- 目标：跳过沟。
- 教学：打开控制面板，把“跳跃”绑定到任意可用键。
- 配置：WASD + 1 个额外槽位；无魔键。
- 设计要点：第一次体验“改键位 = 解谜”。

### 关卡 2：重物挡路
- 目标：把箱子推开。
- 教学：把“推动”绑定到某个键。
- 配置：WASD + 2 个额外槽位；无魔键。
- 设计要点：引入能力组合概念。

### 关卡 3：魔键初现
- 目标：跳过一条较宽的沟。
- 教学：空格被魔化。玩家可以选择：
  - 把跳跃绑到空格（强化跳跃，能跳过宽沟，但按空格加腐败）。
  - 把跳跃绑到普通键（正常跳跃，但需要找其他方式过沟，例如先推箱子垫脚）。
- 配置：WASD + 2 个额外槽位；魔键 = 空格。
- 设计要点：第一次让玩家自己做“是否使用魔键”的决策。

### 关卡 4：双刃剑
- 目标：推动多个箱子搭桥。
- 教学：空格仍是魔键。玩家可以把“推动”绑到空格，获得范围推动；也可以把“推动”绑到普通键，一个一个推。
- 配置：WASD + 3 个额外槽位；魔键 = 空格。
- 设计要点：展示“同一能力的安全版 vs 强化版”的槽位取舍。

### 关卡 5：综合
- 目标：结合跳跃、推动、拉动通过复杂地形。
- 配置：WASD + 3 个额外槽位；魔键 = E。
- 设计要点：魔键变化，玩家必须重新规划。

## 6. 关卡纸面原型 (Paper Prototype)

### 房间 1：跳跃沟
```
[起点] ======= [沟] ======= [出口]
```
- 魔键：空格。
- 解法 A：把跳跃绑到普通键，找箱子垫脚后跳过（安全但慢）。
- 解法 B：把跳跃绑到空格（强化跳跃），直接跳过（快但加腐败）。

### 房间 2：双箱子桥
```
[起点] [箱子A] [箱子B] [高台] [出口]
```
- 魔键：空格，绑定“推动”可获得范围推动。
- 解法 A：把推动绑到空格，一次推多个箱子搭桥（加腐败）。
- 解法 B：把推动绑到普通键，逐个推箱子（慢但安全）。
- 槽位限制：移动 + 跳跃 + 推动（+ 可选魔键版推动）= 4 槽位。

### 房间 3：旋转平台
```
[起点] ——[旋转平台]—— [出口]
              |
         [魔键开关]
```
- 魔键：E，绑定“旋转”可一键把平台转到正确角度。
- 解法 A：把旋转绑到普通键，慢慢调整平台角度。
- 解法 B：把旋转绑到 E，瞬间到位但加腐败。

## 8. 边界情况 (Edge Cases)

- 玩家未绑定移动能力 → 配置阶段强制提示。
- 玩家把所有能力都绑在魔键上 → 允许，但腐败会快速累积。
- 玩家把移动绑在魔键上 → 允许，但每一步都加腐败，属于自虐挑战。
- 玩家在战斗中打开配置菜单 → 本设计建议**战斗中不可打开配置菜单**，配置阶段在关卡开始前完成；时间从进入关卡开始计算，保证评分公平。
- 多魔键关卡中玩家把同一能力绑到两个魔键上 → 允许，两个魔键按下都触发强化版并各自加腐败。
- 玩家用时超过严格时间 → 不失败，但时间奖励为 0，无法获得时间星。
- 玩家吃满槽位通关 → 允许，但无法获得精简星；若同时腐败也爆表，则最多只能拿通关星。
- 玩家尝试刷时间星 → 鼓励使用魔键走捷径，但需控制腐败；形成“快 vs 稳”的二次优化目标。

## 9. 21 天开发范围 (Scope)

### Must Have（必须有）
- 配置阶段 / 控制面板 UI
- 4 个基础能力：移动、跳跃、推动、拉动
- 魔键与腐败系统
- 评分与时间系统（4 星 + 计时器 + HUD）
- 8–10 个教学到中等难度的房间
- 一个可通关的 Demo 结局

### Nice to Have（加分项）
- 时间减缓、旋转等额外能力
- 按键解锁系统
- 隐藏房间与收集品
- 速通计时与零魔键挑战

### Cut if Necessary（必要时砍掉）
- 多结局
- 过场动画与复杂叙事
- 在线排行榜
